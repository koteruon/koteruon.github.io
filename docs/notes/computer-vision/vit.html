Backbone 架構的演進

Convolution Neural Network​

核心概念​

CNN 是電腦視覺領域最經典、也最長壽的模型架構，核心思想是：

  • 局部感受野（Local Receptive Field）
  • 權重共享（Weight Sharing）
  • 層級式特徵抽取（Hierarchical Feature Learning）

模型會從：

邊緣 → 紋理 → 形狀 → 語意 逐層建立對影像的理解。

為什麼 CNN 很強​

  • 天然的 Inductive Bias
    • 局部性（locality）
    • 平移不變性（translation equivariance）
  • 對小資料集友善
  • 計算效率高
    • 卷積對 GPU / TPU 非常友善
  • 特徵具備空間結構
    • 非常適合 detection、segmentation

CNN 的限制​

  • 全域關係建模能力弱
    • 遠距離像素關係只能靠「堆很多層」
  • 長距依賴學習困難
  • 架構設計高度仰賴人工經驗（kernel size、stride、stage 設計）

這些問題，正是 Transformer 類模型試圖解決的地方。

ViT (Vision Transformer)​

核心想法​

ViT 的關鍵突破在於一句話：

把影像當成「token 序列」，直接用 Transformer 來學

具體做法：

  1. 將影像切成固定大小的 patch（例如 16×16）
  2. 每個 patch → 一個 token
  3. 加上 position embedding
  4. 丟進標準 Transformer Encoder

ViT 和 CNN 的本質差異​

面向CNNViT
特徵學習局部 → 全域一開始就是全域
Inductive Bias強弱
長距依賴不擅長天生強
架構設計任務導向極度簡潔

ViT 不是慢慢「看到整張圖」， 而是 一開始就看整張圖。

ViT 的優勢​

  • Global Self-Attention
    • 任意 patch 之間都能互相關注
  • 可擴展性極強
    • 資料越大、模型越大 → 表現越好
  • 架構簡單、通用性高
    • NLP 的成功經驗可以直接遷移

ViT 的現實問題​

  • 非常吃資料
    • 小資料集表現常輸 CNN
  • 沒有內建多尺度 / 階層結構
  • Attention 計算量隨 token² 成長

這讓純 ViT 在 detection、segmentation 任務上不夠「好用」。

Swin Transformer​

設計動機​

Swin Transformer 的核心目標很明確：

讓 Transformer 像 CNN 一樣「好用在各種視覺任務」

因此它做了兩件關鍵的事：

  1. 引入 Hierarchical 結構
  2. 限制 Attention 的計算範圍

Swin 的關鍵機制​

  1. Window-based Self-Attention
    • 將影像切成小 window
    • 只在 window 內做 attention

      大幅降低計算量

  2. Shifted Window
    • 下一層把 window 平移
    • 讓不同 window 之間能交換資訊

      解決 window 被「切斷」的問題

  3. Patch Merging（階層化）
    • 像 CNN 的 pooling / downsampling
    • 特徵解析度逐層降低、語意逐層提高

三、Swin vs ViT​

面向ViTSwin
Attention 範圍全域區域 + 跨區
階層特徵❌✅
計算效率較低高
Detection / Seg需改架構天生適合

Swin 本質上是「為視覺任務重新設計的 Transformer」
