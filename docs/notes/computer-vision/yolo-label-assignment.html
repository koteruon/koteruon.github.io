YOLO Label Assignment 演進(未完成)

Label assignment 的本質問題是：

在 dense prediction 中，一個 GT 對應大量候選框，誰應該負責學習？

不同策略的差異，在於：

  • 正樣本數量如何決定
  • 是否考慮預測品質
  • 是否支援 NMS-free

1. 固定規則匹配（早期 Anchor-based）​

IoU Threshold Matching（YOLOv2–YOLOv5）​

  • 核心機制

    • 計算 anchor 與 GT 的 IoU
    • IoU > threshold → 正樣本
    • IoU < threshold → 負樣本
    • 或選擇 IoU 最大的 anchor 作為負責者

  • 問題

    • 正樣本數量固定，無法依據場景調整

    • 在密集場景中容易出現：

      • 正樣本不足
      • 或多個 anchor 競爭同一物體

    • 不考慮分類與定位品質

本質問題

固定 IoU 規則無法反映模型當前的預測能力，匹配結果往往次優。

動態分配：根據預測品質決定正樣本​

2. OTA（Optimal Transport Assignment）— YOLOX​

來源：YOLOX 論文

  • 核心想法

    • 將 GT 與候選框的匹配建模為：

      → 最優傳輸問題（Optimal Transport）

  • 匹配成本包含

    • Classification loss
    • IoU / regression cost

  • 效果

    • 每個 GT 的正樣本數量 自動決定
    • 匹配更符合當前預測品質
    • Dense scene 表現更穩定

3. SimOTA（YOLOX / YOLOv6 / YOLOv7）​

OTA 的高效近似版本（實務主流）

流程​

  1. 篩選 GT 中心附近的候選點（center prior）
  2. 計算每個候選與 GT 的 IoU
  3. 根據 IoU 排序
  4. 為每個 GT 選擇 Top-k 正樣本
  5. k 由 IoU 動態決定（dynamic k）

優點​

  • 計算量遠低於 OTA
  • 訓練穩定
  • Dense scene 表現好
關鍵

正樣本數量不再固定，而是由匹配品質決定

任務對齊：分類與定位一起考慮​

4. Task-Aligned Assignment（TAL）— YOLOv8​

YOLOv8 採用的核心策略

匹配分數​

score=pclsα⋅IoUβ\text{score} = p_{\text{cls}}^{\alpha} \cdot \text{IoU}^{\beta}score=pclsα​⋅IoUβ

  • 同時考慮：

    • 分類置信度
    • 定位品質

  • 避免出現：

    • 分類高但定位差
    • 或 IoU 高但分類錯誤的正樣本

設計意義​

  • 與 Decoupled Head 一致

  • 讓模型學習：

    好的預測應同時具備分類與定位品質

關鍵

TAL 的核心是讓 classification 與 localization 的學習目標對齊

End-to-End 的關鍵：One-to-One Matching​

5. One-to-One Assignment（YOLOv10）​

NMS-free 的核心需求：

若一個物體對應多個預測，推論仍會產生重複框

因此需要：

每個 GT 只對應一個預測

Consistent Dual Assignments（YOLOv10）​

訓練階段同時使用兩種分配：

One-to-many​

  • 每個 GT 對應多個正樣本
  • 提升訓練穩定性與收斂速度

One-to-one​

  • 每個 GT 僅對應一個最佳預測
  • 學習「唯一匹配」關係

推論階段：

  • 只保留 one-to-one head
  • 直接輸出最終結果（無需 NMS）
關鍵

NMS-free 的本質 = 在訓練中學會唯一匹配
